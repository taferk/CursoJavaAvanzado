Ejercicio 2 - Creación de una aplicación Web

Resumen del Ejercicio 2

Curso: Programación orientada a objetos - Java avanzado

Autores: Jorge Molinero Muñoz, Miguel Sallent Sánchez

Objetivo

Ampliar el ejercicio anterior para que la clase Cliente gestione una lista de Pedidos.

Flujo de datos HTTP

  1. Recoger datos con request.getParameter.
  2. Procesar la información.
  3. Guardar datos con request.setAttribute.
  4. Redirigir con forward al JSP.
  5. Acceder a los datos en el JSP con ${...}.

Pasos principales

  • Paso 1: Añadir clase Pedido y modificar Cliente.
  • Paso 2: Adaptar ClienteDao con patrón Singleton.
  • Paso 3: Actualizar cliente sin perder pedidos.
  • Paso 4: Añadir enlace “Pedidos” en listado de clientes.
  • Paso 5: Crear PedidoController y vista listadoPedidos.jsp.
  • Paso 6: Implementar inserción de pedidos con pedido.jsp.
  • Paso 7: Realizar pruebas con múltiples clientes y pedidos.
  • Paso 8 (Opcional): Validar formularios con Bean Validation.

Validaciones recomendadas

  • Campos obligatorios.
  • Formato correcto de fecha y precio.
  • Validación en servidor para mayor seguridad.

Para más información sobre Bean Validation, se recomienda consultar la documentación oficial de Jakarta Bean Validation API.
